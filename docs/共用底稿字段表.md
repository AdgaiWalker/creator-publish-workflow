# 共用底稿字段表

这份字段表是整条工作流的共用底稿。

它的作用不是直接替某个平台写文案，而是先把所有平台都能复用的核心信息整理出来，再交给不同平台的适配器去改写。

你可以把它理解成：

- `项目门面生成器` 负责尽量把这张表补齐
- `X / 小红书 / 抖音 / B站 / 知乎` 负责拿这张表做平台适配

## 字段表 v1

| 字段 | 作用 | 必填 |
| --- | --- | --- |
| `作品名` | 统一名称，所有平台都要用 | 是 |
| `一句话定位` | 用一句话说清这是什么 | 是 |
| `为什么做` | 创作者动机，适合 README 和发帖 | 是 |
| `解决什么问题` | 说清作品的实际价值 | 是 |
| `对谁有用` | 目标读者或目标用户 | 是 |
| `核心亮点` | 3 个以内最重要的卖点 | 是 |
| `使用方式` | 安装、调用或进入方式 | 是 |
| `主链接` | 仓库、主页或文档页中的主入口 | 是 |
| `推荐截图` | 最适合对外展示的图 | 是 |
| `当前状态` | 已完成、迭代中、首次开源、新版本等 | 否 |
| `最想强调的一句话` | 这次传播最想让别人记住什么 | 是 |
| `当前发布目标` | 这次是想传播、引流、收反馈，还是介绍更新 | 是 |

## 怎么用

最稳的顺序是：

1. 先用 [项目门面生成器](../项目门面生成器/README.md) 整理 README、安装方式和门面信息
2. 把这份字段表补齐
3. 再交给不同平台的适配器去改写
